# Configuration

<Warning>
  Please note that the configuration functionality is currently under construction and some
  sections may be incomplete or subject to changes.
</Warning>

You can configure `@eslint-react/eslint-plugin` by adding a `eslintReact` property to the `settings` field of your ESLint configuration.

### `jsx.pragma` - `string`

(default: `"React"`)

Pragma to use.

### `jsx.fragment` - `string`

(default: `"Fragment"`)

Fragment to use.

### `jsx.extensions` - `string[]`

(default: `[".jsx", ".tsx"]`)

An Array of file extensions to be considered as JSX files.

### `react.version` - `string`

(default: `"detect"`)

React version to use, `"detect"` means auto detect React version from the project's dependencies.

### `reactHooks.alias` - `Record<string, string[]>`

(default: `{}`)\
(e.g. `{ useState: ["useLocalStorageState"] }`)

An record of alias of React's built-in Hooks, ESLint React will treat them as same as the built-in Hooks.

## Example

<ESLintConfig>

<Config>

```js filename=".eslintrc.js"
// ...
module.exports = {
  // ...
  settings: {
    eslintReact: {
      jsx: {
        pragma: "React",
        fragment: "Fragment",
        extensions: [".tsx"]
      },
      react: {
        version: "detect"
      },
      reactHooks: {
        alias: {
          useState: ["useLocalStorageState"],
          // ...
        }
      }
    }
  }
  // ...
}
```

</Config>
