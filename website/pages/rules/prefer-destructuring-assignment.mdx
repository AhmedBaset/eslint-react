# prefer-destructuring-assignment

## Rule category

Style.

## What it does

Enforces the use of destructuring assignment over property assignment.

## Examples

This rule aims to enforce the use of destructuring assignment over property assignment.

### Failing

```tsx {3}
function MyComponent(props) {
  // Warning: Use destructuring assignment
  const items = props.items;

  return <div>{items}</div>;
}
```

```tsx {3}
function MyComponent(props) {
  // Warning: Use destructuring assignment
  return <div>{props.items}</div>;
}
```

### Passing

```tsx
function MyComponent(props) {
  const { items } = props;

  return <div>{items}</div>;
}
```

```tsx
function MyComponent({ items }) {
  return <div>{items}</div>;
}
```

```tsx
function MyComponent({ items }: { items: string[] }) {
  return <div>{items}</div>;
}
```

```tsx
function MyComponent({ items, ...rest }) {
  return <div {...rest}>{items}</div>;
}
```
