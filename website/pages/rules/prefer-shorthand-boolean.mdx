# prefer-shorthand-boolean

## Rule category

Style.

## What it does

Enforces the use of shorthand syntax for boolean attributes.

## Examples

This rule enforces the use of shorthand syntax for boolean attributes.

### Failing

```tsx {2}
// Error: Use shorthand boolean syntax
<MyComponent hidden={true} disabled={false} />;
```

### Passing

```tsx
<MyComponent hidden disabled />;
```
